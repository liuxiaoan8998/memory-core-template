# Memory Core 使用手册

## 日常使用场景

### 场景1: 记录重要信息

**用户**: "记住这个：花哥喜欢穿花衬衫，所以叫花哥"

**Agent 行为**:
1. 检测到触发词"记住这个"
2. 判断为"用户偏好"
3. 写入 `memory/preferences/user-preferences.md`
4. 更新 `MEMORY.md` 中的用户关键信息
5. 确认: "已永久记忆：花哥因喜欢穿花衬衫而得名"

### 场景2: 记录踩坑经验

**用户**: "刚才那个 API 调用失败了，原因是权限没配置对"

**Agent 行为**:
1. 检测到"踩坑"信号
2. 询问: "需要记录这个教训吗？"
3. 用户确认后
4. 写入 `memory/lessons/api-permission-error.md`
5. 包含: 问题描述、原因、解决方法

### 场景3: 重大决策

**用户**: "我们决定用飞书而不是钉钉来做通知"

**Agent 行为**:
1. 检测到"决定"信号
2. 写入 `memory/decisions/feishu-vs-dingtalk.md`
3. 包含: 决策时间、选项对比、最终选择、原因
4. 更新 `MEMORY.md` 关键决策索引

### 场景4: 项目状态更新

**用户**: "小红书首篇笔记发布了"

**Agent 行为**:
1. 检测到项目里程碑
2. 更新 `memory/projects/xiaohongshu-job-account.md`
3. 更新 `NOW.md` 当前优先级
4. 生成新的待办事项

## 记忆检索

### 方式1: 直接询问

**用户**: "我之前说过什么关于飞书授权的？"

**Agent 行为**:
1. 搜索 `memory/lessons/` 和 `memory/decisions/`
2. 找到相关内容
3. 引用并总结

### 方式2: 主动推送

**Agent 开场白**:
```
"花哥，今天聊之前，我看到你昨天提到的 Memory Core 评估，
有个相关教训：反思到行动的转化断裂。需要我先把关键待办推给你吗？"
```

### 方式3: 上下文感知

**用户**: "最近没进展"

**Agent 行为**:
1. 触发 `memory/lessons/stagnation-period-management.md`
2. 启动"重启协议"
3. 建议最小可行行动

## 待办管理

### 查看今日待办

**用户**: "今天有什么待办"

**Agent**:
1. 读取 `memory/NOW.md`
2. 列出当前优先级
3. 询问: "昨天反思的5项任务完成了吗？"

### 添加待办

**用户**: "记住待办：明天要联系客服"

**Agent**:
1. 添加到 `memory/NOW.md`
2. 确认优先级
3. 设置提醒（如集成日历）

### 完成待办

**用户**: "客服联系完了"

**Agent**:
1. 在 `memory/NOW.md` 标记完成
2. 记录到今日日志
3. 询问是否需要提取经验教训

## 反思与复盘

### 每日自动反思

系统每天 01:00 自动生成深度反思，包含:
- 今日核心主题
- 深度洞察
- 明日行动清单
- 知识提取建议

### 手动触发反思

**用户**: "帮我复盘一下这周的工作"

**Agent**:
1. 读取本周每日日志
2. 分析模式
3. 生成反思报告
4. 提取知识到 lessons/

### 查看历史反思

**用户**: "看看上周的反思"

**Agent**:
1. 列出 `memory/reflections/` 文件
2. 按时间排序
3. 提供摘要

## 知识库维护

### 查看知识库健康度

**用户**: "记忆系统健康吗"

**Agent**:
1. 读取 `memory/INDEX.md`
2. 读取 `memory/NOW.md` 查看当前待办
3. 报告统计:
   - 总文件数
   - 各分类数量
   - 过时文件数
   - 活跃度指标

### 清理过时记忆

**用户**: "清理一下旧记忆"

**Agent**:
1. 扫描 >30天未更新的文件
2. 询问是否归档或删除
3. 执行清理
4. 更新索引

### 搜索特定记忆

**用户**: "找找关于小红书的记忆"

**Agent**:
1. 搜索文件名和内容
2. 返回匹配结果
3. 提供摘要和链接

## 故障排查

### 记忆没有记录

**检查**:
1. 触发词是否正确（"记住这个"/"一直记住"）
2. 文件权限是否正常
3. 磁盘空间是否充足

### 记忆检索不到

**检查**:
1. 文件是否在正确的分类目录
2. INDEX.md 是否已更新
3. 搜索关键词是否匹配

### 反思没有生成

**检查**:
1. OpenClaw cron 是否配置正确
2. gateway 是否运行
3. 日志是否有错误

## 最佳实践

### 1. 及时记录

- 踩坑后立即记录
- 决策后立即记录
- 不要依赖"稍后整理"

### 2. 定期复盘

- 每周回顾 lessons/
- 每月更新 INDEX.md
- 每季度清理过时内容

### 3. 主动使用

- 对话开始时引用相关记忆
- 遇到相似问题时调用经验教训
- 定期查看 `memory/NOW.md`

### 4. 保持精简

- MEMORY.md <5KB
- 每日日志只记录实质内容
- 及时提取知识到分类目录

## 示例对话

### 完整示例1: 记录 + 检索 + 应用

```
用户: 刚才那个 API 调用失败了，原因是权限没配置对

Agent: 需要记录这个教训吗？这样下次遇到类似问题可以快速解决。

用户: 好

Agent: 已记录到 lessons/api-permission-error.md。下次遇到 API 问题时，
我会提醒你检查权限配置。

---

[3天后]

用户: 这个 API 又出问题了

Agent: 记得吗？3天前你遇到过类似问题，原因是权限配置。
建议先检查：1. token 是否过期 2. scope 是否正确

用户: 对，是 token 过期了

Agent: 需要更新这个教训，添加"token 过期"的情况吗？
```

### 完整示例2: 决策 + 追踪

```
用户: 我们决定用飞书而不是钉钉

Agent: 已记录为重大决策。原因是什么？这样以后可以复盘。

用户: 因为飞书文档协作更好

Agent: 已记录到 decisions/feishu-vs-dingtalk.md。

---

[1个月后]

用户: 看看我们之前为什么选飞书

Agent: 根据 decisions/feishu-vs-dingtalk.md：
决策时间：2026-03-XX
原因：飞书文档协作更好
当前状态：使用中
需要更新状态吗？
```
