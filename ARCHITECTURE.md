# Memory Core 系统架构详解

## 设计哲学

### 核心原则

1. **记录 ≠ 记忆**
   - 记录是存储，记忆是提取和应用
   - 系统不仅要存，还要能主动推送

2. **自动化 + 人工筛选**
   - 自动化降低记录门槛
   - 人工/AI 筛选保证质量

3. **渐进式披露**
   - 核心记忆精简（<5KB）
   - 详细内容分层存储
   - 按需加载，避免信息过载

## 核心组件详解

### 1. MEMORY.md（核心记忆）

**作用**: Agent 每次会话首先加载的记忆

**内容**:
- 用户关键信息（姓名、角色、偏好）
- 当前优先级
- 重要配置
- 关键决策索引
- 经验教训索引

**约束**: <5KB，只保留最核心信息

**更新触发**:
- 用户说"一直记住"/"永久记忆"
- 重大决策确认
- 核心偏好变更

### 2. memory/NOW.md（当前待办）

**作用**: 当前优先级和待办事项

**位置**: `memory/NOW.md`（不在根目录）

**内容**:
- 当前优先级列表
- 项目状态追踪
- 近期待办

**更新频率**: 每日更新，或任务状态变化时

### 3. HEARTBEAT.md（心跳扫描）

**作用**: 每小时执行的记忆维护任务

**执行内容**:
1. 扫描最近60分钟会话
2. 清洗消息（过滤低价值内容）
3. 记录实质内容到每日日志
4. 提取知识到分类目录
5. 更新索引

**输出**:
- 每日日志更新
- 知识库文件更新
- 索引更新

### 4. 夜间反思（Nightly Reflection）

**两阶段执行**:

**阶段1: 素材生成（00:45）**
- 扫描过去24小时会话
- 提取关键决策、洞察、教训
- 生成反思素材

**阶段2: 深度反思（01:00）**
- 分析今日核心主题
- 提取深度洞察
- 识别可复用方法论
- 生成明日行动清单
- 提取知识到 lessons/ 或 decisions/

### 5. 知识库分类

#### lessons/（经验教训）
- **标准**: 踩坑 + 解决方法（下次能直接用）
- **内容**: 问题描述、案例、根因、解决方法、关键教训
- **更新**: 反思时提取，或遇到新问题时

#### decisions/（重大决策）
- **标准**: 影响后续方向的重要决定
- **内容**: 决策时间、背景、选项、最终决策、原因、执行计划、状态
- **更新**: 重大决策确认时

#### projects/（项目状态）
- **标准**: 正在进行的项目追踪
- **内容**: 项目目标、当前状态、里程碑、待办、风险
- **更新**: 项目状态变化时

#### preferences/（用户偏好）
- **标准**: 用户偏好配置
- **内容**: 沟通风格、工作习惯、偏好设置
- **更新**: 用户明确偏好时

#### reflections/（深度反思）
- **标准**: 周期性深度反思记录
- **内容**: 反思周期、核心主题、深度洞察、明日行动
- **更新**: 每天01:00自动生成

## 数据流

### 写入流程

```
用户对话
    ↓
触发词检测（"记住这个"）
    ↓
内容分类（ lessons/decisions/projects/preferences ）
    ↓
写入对应文件
    ↓
更新 INDEX.md 索引
```

### 读取流程

```
会话开始
    ↓
加载 MEMORY.md（核心记忆）
    ↓
加载 NOW.md（当前待办）
    ↓
上下文感知检索（按需加载 lessons/decisions 等）
    ↓
主动推送相关记忆
```

## 关键设计决策

### 决策1: 文件系统 vs 数据库

**选择**: 文件系统（Markdown）

**原因**:
- 人类可读，便于直接查看
- 版本控制友好（Git）
- Agent 可直接读写
- 无需外部依赖

**权衡**:
- 查询效率不如数据库
- 需要索引机制
- 文件数量增长需管理

### 决策2: 主动推送 vs 被动检索

**当前**: 以被动检索为主

**问题**: 用户可能不知道记忆存在

**改进方向**: 主动推送机制
- 对话开场引用相关记忆
- 上下文感知自动触发
- 每日记忆摘要

### 决策3: 自动化程度

**选择**: "自动化捕获 + AI 筛选"

**自动化层**:
- 心跳扫描（每小时）
- 夜间反思（每天）
- 索引维护（自动）

**人工/AI层**:
- 内容分类判断
- 知识价值评估
- 反思深度提炼

## 性能考虑

### 文件数量管理

- 每日日志：保留30天，自动归档
- 深度反思：保留90天，重要反思提取到 lessons/
- 索引定期压缩

### 加载优化

- MEMORY.md 保持 <5KB
- 按需加载其他文件
- 缓存常用记忆

## 安全与隐私

- 敏感信息不写入共享记忆
- 用户可删除特定记忆
- 定期备份机制
